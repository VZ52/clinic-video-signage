# 06. Запасной путь: обычный телевизор с флешкой (без автоматизации)

Этот документ — про точки, которые **нельзя** подключить к центральной системе:

- обычный телевизор (не Smart), **и**
- нет интернета на точке, **и**
- нет возможности/бюджета поставить приставку.

Такой телевизор физически не может получать видео по сети. Для него остаётся
**ручная замена флешки**. Ниже — как сделать этот процесс максимально аккуратным,
чтобы контент не «расползался» и совпадал с тем, что показывает остальная сеть.

> 📌 Стратегически: старайтесь сокращать число таких точек. Как только на точке
> появляется интернет — переводите её на Android-приставку
> ([05-players-android-box.md](05-players-android-box.md)) и она входит в общую
> автоматизацию.

---

## Организационная схема

Держим «эталонные» наборы видео централизованно и раскладываем их на флешки перед
выездом. Идея — чтобы флешечная точка показывала **то же самое**, что видит её
группа в системе.

```
  Центральное хранилище видео (папка на сервере / в облаке)
  ├── ОБЩЕЕ/            ← то, что играет у группы «Все»
  ├── ДЕТСКОЕ/          ← то, что играет у группы «Детские»
  └── СТОМАТОЛОГИЯ/     ← и т.д. по группам
             │
             ▼  копируем нужную папку на флешку
        [ Флешка для точки X ]
             │
             ▼  выезд/пересылка на точку
        [ Обычный ТВ: воткнуть флешку ]
```

---

## Как подготовить флешку

1. **Формат флешки — FAT32 или exFAT.** Большинство телевизоров читают эти
   форматы. Если файлы больше 4 ГБ — нужен exFAT (FAT32 не хранит файлы >4 ГБ).
2. **Формат видео — H.264 (MP4), 1080p.** Самый совместимый вариант для ТВ.
   Рекомендации по кодированию — [07-content-and-playlists.md](07-content-and-playlists.md).
3. **Имена файлов — по порядку воспроизведения.** Многие ТВ проигрывают по алфавиту,
   поэтому нумеруйте: `01_ролик.mp4`, `02_ролик.mp4`, ...
4. **Проверьте автоповтор.** В настройках USB-плеера телевизора включите
   «Повтор всех» / «Repeat All», чтобы список крутился по кругу.

## Журнал точек (чтобы не путаться)

Заведите простую таблицу (в этом репозитории или в общей табличке). Пример:

| Точка | Тип | Группа контента | Кто обслуживает | Последняя замена |
|-------|-----|-----------------|-----------------|------------------|
| Клиника-Юг, холл | Флешка | ОБЩЕЕ | Иванов | 2026-07-01 |
| Клиника-Юг, детское | Флешка | ДЕТСКОЕ | Иванов | 2026-07-01 |
| Клиника-Центр, регистратура | Приставка (авто) | Все | — | авто |

Так вы всегда знаете, куда какую флешку везти и когда в последний раз меняли.

---

## Регламент обновления «флешечных» точек

1. Обновили видео в центральном хранилище (папки ОБЩЕЕ/ДЕТСКОЕ/…).
2. Записали свежие флешки для каждой ручной точки согласно её группе.
3. Подписали флешки (точка + дата).
4. Отвезли/передали, воткнули, проверили, что играет и повторяется по кругу.
5. Отметили дату в журнале.

---

## Почему нельзя «просто автоматизировать флешку»

Чтобы устройство скачивало видео из интернета и подменяло его на флешке, ему
нужен процессор, ОС и сеть — то есть оно уже перестаёт быть «просто флешкой» и
становится приставкой. Поэтому единственные два честных варианта для обычного ТВ:

- **есть сеть** → Android-приставка и полная автоматизация;
- **нет сети** → ручная флешка по регламенту выше.

Промежуточного «волшебного» варианта не существует — на это важно не тратить
время при внедрении.
