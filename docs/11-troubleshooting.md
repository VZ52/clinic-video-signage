# 11. Решение проблем

Типовые неполадки и что делать. Ищите свой симптом.

---

## Плеер / экран

### Новый плеер не появляется в списке Displays
- Проверьте на плеере **CMS Address** — точный адрес сервера (со `https://` или
  `http://`), без опечаток.
- Проверьте **Key** — он должен совпадать с **CMS Secret Key** в
  Administration → Settings → Displays.
- Проверьте интернет на точке: откройте адрес CMS в браузере устройства — панель
  должна открыться.
- Firewall/провайдер точки может блокировать сервер — проверьте, что порт 80/443
  доступен с этой точки.

### Экран есть в списке, но статус офлайн (серый/красный)
- Устройство выключено, нет питания или интернета на точке.
- Плеер давно не выходил на связь — посмотрите «last accessed» у экрана.
- Перезагрузите плеер/приставку. Проверьте сеть.
- Если недавно менялся адрес/домен CMS — плеер ходит по старому адресу; поправьте
  настройки плеера.

### Экран онлайн, но показывает заставку Xibo / пусто
- Экрану не назначен контент. Проверьте, что он в группе `Все` и что на эту группу
  есть событие в **Schedule** (см. [08-groups-and-scheduling.md](08-groups-and-scheduling.md)).
- Раскладка не опубликована (в черновике) — откройте Layout и нажмите **Publish**.
- Контент ещё не скачался — нажмите **Collect Now** или подождите опрос.

### Экран показывает НЕ то видео
- Проверьте, в каких он **группах** (Displays → Edit → Display Groups).
- Проверьте **приоритеты** событий в расписании: раскладка с более высоким
  приоритетом перекрывает низкую. Возможно, «исключение» назначено шире, чем
  нужно.
- Классика: детский ролик назначен на группу, куда случайно попал обычный экран.

### Видео тормозит, рвётся, чёрный экран со звуком
- Файл слишком тяжёлый для этого плеера. Пережмите в H.264 1080p с умеренным
  битрейтом (см. [07-content-and-playlists.md](07-content-and-playlists.md)).
- Кодек H.265/HEVC или экзотический контейнер — не все плееры тянут. Используйте
  MP4/H.264.
- Слабая приставка — замените на модель помощнее.

### После отключения питания экран не вернулся к видео
- Не настроен автозапуск. См. [05-players-android-box.md](05-players-android-box.md),
  шаг 4 (автозапуск приставки, автозапуск Xibo Player, включение ТВ по HDMI-CEC).

## Сервер

### Панель не открывается в браузере
- Проверьте, что контейнеры подняты:

  ```bash
  cd /opt/clinic-video-signage
  docker compose ps
  ```

  Все должны быть `Up`. Если нет — `docker compose up -d` и смотрите логи.
- Смотрите логи веб-контейнера:

  ```bash
  docker compose logs --tail=100 cms-web
  ```

- Порт 80 занят другим сервисом? Проверьте `sudo ss -tlnp | grep :80`.

### При первом запуске долго «крутит» / ошибки миграции БД
- Первая инициализация БД занимает несколько минут — дайте время.
- Если в логах ошибка подключения к БД — почти всегда неверный/пустой
  `MYSQL_PASSWORD` в `config.env`. Задайте пароль (буквы+цифры, 16 символов) и
  пересоздайте:

  ```bash
  docker compose down
  # ВНИМАНИЕ: если БД уже создалась со старым паролем, а вы меняете пароль,
  # проще на чистой установке удалить shared/db и начать заново (данных ещё нет).
  docker compose up -d
  ```

### Кончается место на диске
```bash
df -h
du -sh /opt/clinic-video-signage/shared/*
```
- Разрослась `library` — удалите неиспользуемые ролики через **Library** в панели.
- Много старых бэкапов в `shared/backup` или `backups/` — почистите старые архивы.

### Не грузится большой видеофайл (>2 ГБ)
- Увеличьте лимиты в `config.env`:
  ```
  CMS_PHP_POST_MAX_SIZE=4G
  CMS_PHP_UPLOAD_MAX_FILESIZE=4G
  CMS_PHP_MAX_EXECUTION_TIME=600
  ```
  и примените: `docker compose up -d`.

### «Collect Now» не срабатывает мгновенно
- Проверьте, что порт **9505** (XMR) доступен плеерам снаружи.
- Проверьте адрес XMR в Administration → Settings.
- Даже без XMR плеер обновится сам при следующем опросе — просто не мгновенно.
