# 01. Что это и зачем

## Проблема, которую решаем

У сети клиник есть телевизоры, которые крутят рекламные/информационные ролики.
Клиники разбросаны географически. Сейчас видео меняют вручную — ходят по точкам
и переставляют флешки. Это долго, неудобно и легко забыть какую-то точку.

**Хочется так:** загрузил новое видео у себя за компьютером — и оно само
приехало почти на все телевизоры. А для отдельных мест (детские отделения и т.п.)
можно назначить своё, отличное от остальных, видео.

## Что делает эта система

- **Один центр управления.** Веб-панель, куда вы заходите из браузера и грузите
  видео. Больше никуда ходить не надо.
- **Автоматическая доставка.** Телевизоры-плееры сами скачивают новое видео из
  центра через интернет и начинают его показывать.
- **Разделение по точкам.** «Всем — ролик А, детским отделениям — ролик Б,
  стоматологии — ролик В». Настраивается через *группы экранов*.
- **Устойчивость к обрывам сети.** Плеер хранит локальную копию, поэтому при
  пропаже интернета продолжает играть последнее скачанное видео.
- **Мониторинг.** В панели видно, какой телевизор онлайн, что он сейчас
  показывает, когда последний раз выходил на связь.

## Из чего состоит

1. **Сервер Xibo CMS** — «мозг» системы. Одна установка на весь проект.
   Разворачивается через Docker (см. [03-server-install.md](03-server-install.md)).
2. **Плееры** — то, что стоит на точках и показывает видео на телевизоре:
   - **Smart TV** → устанавливаем приложение Xibo прямо на телевизор
     ([04-players-smart-tv.md](04-players-smart-tv.md));
   - **Обычный телевизор + Android-приставка** → плеер живёт на приставке,
     телевизор просто показывает картинку по HDMI
     ([05-players-android-box.md](05-players-android-box.md));
   - **Обычный телевизор без интернета** → запасной путь, ручная флешка
     ([06-usb-fallback.md](06-usb-fallback.md)).

## Ключевые понятия (коротко)

| Термин | Простыми словами |
|--------|------------------|
| **CMS** | Сервер с веб-панелью — центр управления |
| **Display (Экран)** | Один зарегистрированный плеер = один телевизор |
| **Display Group (Группа экранов)** | Набор телевизоров, которым удобно назначать контент разом |
| **Media (Медиа)** | Загруженный файл — например видеоролик |
| **Layout (Раскладка)** | Что показывается на экране: одно видео на весь экран, или видео + бегущая строка и т.п. |
| **Playlist (Плейлист)** | Список медиа, играющих по очереди |
| **Schedule (Расписание)** | Правило «какой раскладке на какой группе и когда играть» |
| **Campaign (Кампания)** | Несколько раскладок, объединённых для показа по очереди |

Полный словарь: [12-glossary.md](12-glossary.md).

## Почему Xibo, а не облачный сервис

Есть облачные SaaS-сервисы (Yodeck, OptiSigns, ScreenCloud). Они удобны, но берут
**абонплату за каждый экран** — обычно 8–15 $/мес. На 50–200 экранах это
400–3000 $ **каждый месяц, постоянно**.

Xibo CMS — **бесплатный и открытый**, ставится на свой сервер. Платить надо
только за:
- сервер (недорогой VPS или свой компьютер/сервер);
- приставки там, где стоят обычные телевизоры (разовая покупка).

Минус подхода — сервер надо один раз развернуть и потом обслуживать (бэкапы,
обновления). Именно для этого написана вся документация в этом репозитории.
