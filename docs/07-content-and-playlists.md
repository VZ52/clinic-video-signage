# 07. Загрузка видео и сборка плейлистов

Здесь — как залить ролик в систему и «упаковать» его так, чтобы плееры могли
показывать. В Xibo это две сущности:

- **Media (Медиа)** — сам файл (видео).
- **Layout (Раскладка)** — то, что реально видно на экране: как правило, одно
  видео на весь экран (может быть и сложнее: видео + бегущая строка + логотип).

Раскладки потом назначаются на группы экранов через расписание
([08-groups-and-scheduling.md](08-groups-and-scheduling.md)).

---

## Часть A. Подготовка видеофайла

Чтобы видео гладко играло на дешёвых плеерах, приведите его к «дружелюбному»
формату:

| Параметр | Рекомендация |
|----------|--------------|
| Контейнер | **MP4** |
| Видеокодек | **H.264** (H.265/HEVC не все плееры тянут) |
| Разрешение | **1920×1080** (Full HD), если экран не 4K |
| Кадры | 25 или 30 fps |
| Битрейт | 4–8 Мбит/с (баланс качества и размера) |
| Аудиокодек | **AAC** (если звук нужен) |

Пример пережатия через бесплатный **ffmpeg**:

```bash
ffmpeg -i исходник.mov -c:v libx264 -preset medium -crf 22 \
       -vf "scale=1920:1080" -c:a aac -b:a 128k результат.mp4
```

Чем меньше файл — тем быстрее он раздаётся на все точки и меньше нагрузка на
интернет клиник.

---

## Часть B. Загрузить видео в CMS

1. В веб-панели откройте **Library** (Библиотека).
2. Нажмите **Add Media / Upload** (Добавить/Загрузить).
3. Выберите видеофайл(ы) и дождитесь окончания загрузки.
4. (Полезно) задайте понятное имя и **теги**, напр. `общее`, `детское`,
   `акция-июль` — по тегам потом легко искать.

> Если файл больше 2 ГБ и не грузится — увеличьте лимит в `config.env`
> (`CMS_PHP_UPLOAD_MAX_FILESIZE`), см. [03-server-install.md](03-server-install.md),
> и перезапустите: `docker compose up -d`.

## Часть C. Собрать раскладку (Layout)

### Вариант 1. Быстрый — одно видео на весь экран

1. **Layouts** → **Add Layout** (Добавить раскладку). Задайте имя, напр.
   `Общее — июль`.
2. Откроется дизайнер. Задайте разрешение раскладки под ваши экраны (обычно
   1920×1080, альбомная).
3. Перетащите на холст виджет **видео** и выберите ваш файл из библиотеки
   (или загрузите новый прямо здесь).
4. Убедитесь, что видео занимает весь регион (растяните на весь экран).
5. **Publish** (Опубликовать) раскладку — до публикации она в черновике и не
   раздаётся.

### Вариант 2. Несколько видео по очереди — через Playlist

Если на экране должно крутиться несколько роликов подряд:

1. В раскладке выделите регион и откройте его **Playlist** (Плейлист).
2. Добавьте в плейлист несколько видео — они будут играть по порядку и
   зациклятся.
3. Порядок можно менять перетаскиванием; для каждого элемента можно задать
   длительность (для видео обычно = длине ролика).
4. **Publish**.

> Плейлисты бывают и «умные» (Dynamic Playlist) — автоматически включают все
> медиа с определённым тегом. Удобно: добавил новому ролику тег `детское` — он
> сам попал в детский плейлист. См. официальный манускрипт Xibo, раздел Playlists.

---

## Часть D. Проверка перед раскаткой на все точки

Прежде чем назначать на 200 экранов, проверьте раскладку на одном:

1. Назначьте раскладку на **один тестовый экран** (или тестовую группу из одного
   плеера) — как это сделать, см. [08-groups-and-scheduling.md](08-groups-and-scheduling.md).
2. Убедитесь, что видео играет, не тормозит, звук/картинка корректны.
3. Только потом назначайте на боевые группы.

Это спасает от ситуации «залил битый файл сразу на всю сеть».
