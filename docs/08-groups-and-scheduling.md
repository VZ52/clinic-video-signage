# 08. ⭐ Группы экранов и расписание: «всем — общее, отдельным — своё»

Это самый важный документ для вашей задачи. Здесь — как сделать, чтобы:

- на **большинстве** телевизоров крутилось общее видео;
- в **детских отделениях** — своё, детское;
- при этом менять контент нужно **в одном месте**, а не бегать по точкам.

Механизм состоит из двух частей: **группы экранов** (кому) и **расписание с
приоритетом** (что и когда).

---

## Идея на пальцах

```
  Экран может входить сразу в несколько групп.
  Расписание назначает раскладку на группу и даёт ей ПРИОРИТЕТ.
  Плеер показывает раскладку с САМЫМ ВЫСОКИМ приоритетом среди своих.

  Группа «Все»        → «Общее видео»     приоритет 0   (фон по умолчанию)
  Группа «Детские»    → «Детское видео»   приоритет 10  (перекрывает фон)

  ТВ в регистратуре   входит в «Все»                  → показывает Общее
  ТВ в детском        входит в «Все» + «Детские»      → показывает Детское
```

Поскольку у детской раскладки приоритет выше, на детских экранах она «побеждает»
общую. На всех остальных экранах детской раскладки нет — там играет общая.

---

## Шаг 1. Создать группы экранов

**Displays → Display Groups → Add Display Group.**

Создайте нужные группы. Рекомендуемый набор для старта:

| Группа | Смысл |
|--------|-------|
| `Все` | Все автоматические экраны сети (общий фон-контент) |
| `Детские` | Только экраны в детских отделениях |
| `Стоматология` | Пример узкой группы под отдельный контент |
| `Тест` | 1 экран для проверки перед раскаткой |

Можно добавлять и «географические» группы (`Клиника-Центр`, `Клиника-Север`), если
захотите таргетировать по клинике целиком.

> 💡 Совет: держите группу `Все` действительно всеобщей — включайте в неё каждый
> новый экран. Специализированные группы (`Детские` и т.п.) — поверх неё.

---

## Шаг 2. Разложить экраны по группам

Для каждого экрана:

1. **Displays** → выберите экран → **Edit**.
2. Вкладка/поле **Display Groups** — отметьте, в какие группы он входит.
3. Сохраните.

Правило:
- **каждый** автоматический экран → в группу `Все`;
- экран в детском отделении → **дополнительно** в `Детские`;
- экран стоматологии → дополнительно в `Стоматология`; и т.д.

Массово удобнее наоборот: открыть **Display Group → Members** и отметить всех её
участников за один раз.

---

## Шаг 3. Назначить контент через расписание

**Schedule → Add Event** (Расписание → Добавить событие).

### 3.1. Общий фон (для всех)

| Поле | Значение |
|------|----------|
| Type (тип) | Layout / Campaign |
| Layout | ваша общая раскладка (напр. `Общее — июль`) |
| Display Group (кому) | **`Все`** |
| Start/End (когда) | «Always» / без ограничения по датам (постоянно) |
| **Priority (приоритет)** | **0** (низкий — это фон) |

Сохраните. Теперь на всех экранах группы `Все` играет общее видео.

### 3.2. Исключение для детских

| Поле | Значение |
|------|----------|
| Layout | детская раскладка (напр. `Детское — июль`) |
| Display Group | **`Детские`** |
| Start/End | постоянно (или на нужный период) |
| **Priority** | **10** (выше, чем у общего) |

Сохраните. Экраны в `Детские` теперь показывают детское видео вместо общего;
все прочие продолжают показывать общее.

### 3.3. Ещё исключения

По той же схеме: раскладка → узкая группа (`Стоматология`, конкретная клиника,
даже один экран) → приоритет выше 0. Можно строить «слои»:

```
приоритет 0   Общее           → группа «Все»
приоритет 10  Детское         → группа «Детские»
приоритет 20  Акция стомат.   → группа «Стоматология»
приоритет 50  Срочное объявл. → конкретная клиника (на время)
```

Чем выше приоритет — тем «главнее» раскладка на тех экранах, где она назначена.

---

## Шаг 4. Проверка логики

Мысленно (или реально по одному экрану из каждой группы) пройдите:

- ТВ только в `Все` → должен показывать **Общее**. ✅
- ТВ в `Все` + `Детские` → должен показывать **Детское**. ✅
- ТВ в `Все` + `Стоматология` → **контент стоматологии**. ✅

Если экран показывает не то — почти всегда причина в том, что он не в той группе,
или приоритеты выставлены неверно. См. [11-troubleshooting.md](11-troubleshooting.md).

## Частые сценарии

### «Поменять видео сразу везде»

Не трогайте группы и расписание. Просто обновите **содержимое общей раскладки/
плейлиста** (замените видео внутри) и опубликуйте. Все экраны группы `Все`
подтянут новое при следующем опросе. Пошагово — [09-daily-usage.md](09-daily-usage.md).

### «В детских — поменять только детское»

Обновите **детскую** раскладку/плейлист. Общих экранов это не коснётся.

### «Добавить новую точку»

Зарегистрируйте плеер ([04](04-players-smart-tv.md)/[05](05-players-android-box.md)),
авторизуйте экран, добавьте его в `Все` (и, если нужно, в спец-группу). Контент
подтянется автоматически — отдельно назначать не надо.

### «Временное объявление на одну клинику»

Создайте раскладку с объявлением, назначьте на группу этой клиники с высоким
приоритетом и датами начала/конца. По окончании периода экраны сами вернутся к
обычному контенту.

### «Разное по времени суток»

В событии расписания можно задать не «постоянно», а конкретные часы/дни. Так,
например, утром одно, вечером другое — на одной и той же группе.

---

## Почему это удобно на 50–200 экранах

- Новый экран достаточно один раз положить в правильные группы — контент
  прилетает сам.
- Контент-логика описана «слоями приоритетов», а не привязана к каждому
  телевизору по отдельности.
- Изменение общего ролика = одно действие для всей сети.
