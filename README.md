# 📺 Система показа видео на телевизорах в клиниках

Централизованное управление видео на телевизорах в сети клиник/отделений,
которые физически разбросаны далеко друг от друга.

Вы загружаете видео **в одном месте** (веб-панель на своём рабочем компьютере) —
и оно **автоматически обновляется на всех телевизорах** через интернет. При этом
можно задавать **разный контент для разных точек**: например, в детских
отделениях показывать одни ролики, а во всех остальных — другие.

Система построена на бесплатном открытом ПО **[Xibo CMS](https://xibosignage.com)**
(это зрелая система «цифровых вывесок», digital signage). Этот репозиторий —
подробная инструкция и готовые файлы для развёртывания и эксплуатации именно под
задачу «сеть клиник».

> ⚠️ **Важно про «просто флешку в телевизоре».** Обычный телевизор, в который
> напрямую воткнута флешка, **невозможно** обновлять по сети — у него нет
> «мозгов», которые скачивали бы новое видео. Чтобы точка обновлялась
> автоматически, на ней должно быть сетевое устройство-плеер: либо **Smart TV**
> (ставим приложение), либо дешёвая **Android-приставка** к обычному телевизору.
> Для точек совсем без интернета остаётся ручная замена флешки — см.
> [docs/06-usb-fallback.md](docs/06-usb-fallback.md).

## 🚀 Быстрый старт (для нетерпеливых)

```bash
# на сервере с Linux и установленным Docker
git clone https://github.com/VZ52/clinic-video-signage.git
cd clinic-video-signage
cp config.env.template config.env
nano config.env               # вписать MYSQL_PASSWORD (16 символов, буквы+цифры)
docker compose up -d          # первый запуск занимает 3-5 минут
```

Открыть в браузере `http://IP-адрес-сервера` → войти (`xibo_admin` / пароль по
умолчанию `password`) → **сразу сменить пароль**.

Дальше — по инструкциям ниже. Полная версия установки:
[docs/03-server-install.md](docs/03-server-install.md).

## 🗺️ Как это работает в двух словах

```
  ВЫ (браузер, рабочее место)
        │  загружаете видео, назначаете на группы экранов
        ▼
  ┌─────────────────────────┐
  │      Xibo CMS           │   ← один сервер (VPS или свой), Docker
  │  веб-панель + база +    │
  │  хранилище видео        │
  └───────────┬─────────────┘
              │  интернет (плееры сами periodically скачивают новое)
   ┌──────────┼───────────────────────┬──────────────────────┐
   ▼          ▼                        ▼                      ▼
[Smart TV]  [Smart TV]         [Приставка + обычный ТВ]  [Обычный ТВ без сети]
 группа      группа                   группа              флешка вручную
 «Все»       «Детские»                «Стоматология»      (запасной путь)
```

- **Центр** — один сервер Xibo CMS, единственное место, куда вы грузите видео.
- **Плееры** сами, по своей инициативе, регулярно опрашивают сервер и скачивают
  новое видео, кэшируя его локально. Поэтому если интернет пропал — видео
  продолжает играть с локальной копии.
- **Разделение контента** — через *группы экранов* и *расписание с приоритетом*.
  Подробно и с примерами: [docs/08-groups-and-scheduling.md](docs/08-groups-and-scheduling.md).

## 💰 Что это стоит

| Статья | Ориентировочно |
|--------|----------------|
| Сервер Xibo CMS (VPS 2 CPU / 4 ГБ RAM) | ~400–700 ₽/мес, либо свой сервер бесплатно |
| Само ПО Xibo CMS | **0 ₽** (open-source) |
| Приложение Xibo для Android/Smart TV | Есть бесплатная лицензия плеера |
| Android-приставка (там, где обычный ТВ) | ~2 000–4 000 ₽ разово за точку |
| Smart TV | ничего не докупаем, ставим приложение |

Сравнение с облачными SaaS-сервисами (Yodeck/OptiSigns и т.п.), где платят
за каждый экран ежемесячно, — см. [docs/01-overview.md](docs/01-overview.md).
